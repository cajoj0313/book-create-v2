## 1. 需求背景

### 问题描述

灵笔项目的世界观构建流程要求用户先输入小说背景描述，才能开始生成。但对于完全没有创作灵感的用户来说，这个"起点"门槛过高：

| 问题 | 影响 | 用户反馈 |
|------|------|----------|
| 用户无灵感时无法输入描述 | 无法开始创作流程 | "不知道写什么" |
| 用户想快速体验系统 | 需要先构思内容 | "只想试试看功能" |
| 用户想探索不同创意 | 无法快速获取灵感 | "想看看能生成什么" |

### 用户痛点

| 痛点 | 场景 | 严重度 |
|------|------|--------|
| 无创作灵感 | 用户想写小说但完全不知道从何开始 | **P1** |
| 快速体验需求 | 新用户想快速体验系统，不想花时间构思 | **P2** |
| 创意探索需求 | 用户想探索不同类型的世界观创意 | **P2** |

### 使用场景

**场景 1**: 用户完全没有创作灵感，想要一个"起点"
- 用户打开世界观构建页面
- 不知道该输入什么描述
- 点击"随机生成"按钮
- 系统直接生成一个完整的都市世界观
- 用户查看结果，作为创作起点

**场景 2**: 用户想快速体验系统功能
- 用户刚注册，想体验完整流程
- 不想花时间构思世界观
- 点击"随机生成"快速体验

## 2. 需求范围定义

### MVP 范围划分

| 优先级 | 功能 | 预计工期 | 说明 |
|--------|------|----------|------|
| **P0（必须）** | 一键随机生成都市世界观 | 2-3 天 | MVP 核心功能 |
| **P1（重要）** | 随机生成提示词供选择 | 2 天 | 第二阶段功能 |
| **P2（次要）** | 支持更多小说类型 | 1 天 | 后续迭代扩展 |

**总工期**: MVP（P0）2-3 天

### MVP 定义

**Must have（本次实现）**:
- 前端新增"随机生成"按钮，与现有"开始生成"按钮并列
- 点击后无需任何输入，直接生成都市类型世界观
- 复用现有的 SSE 流式生成和展示流程
- 复用现有的确认对话框

**Nice to have（后续迭代）**:
- 随机生成提示词供用户选择（第二阶段）
- 支持更多小说类型（武侠、玄幻、科幻等）

**Out of scope（本次不做）**:
- 用户自定义随机生成参数
- 随机生成历史记录
- 随机生成结果评分

## 3. 详细需求清单

### 3.1 前端需求

#### 3.1.1 UI 变更

**位置**: WorldBuilder.tsx 页面

**变更内容**:
- 在"输入小说背景描述"区域下方，新增一个独立的"随机生成"按钮
- 按钮文案："AI 随机生成"
- 按钮样式：与"开始生成"按钮并列，使用不同的颜色区分（建议 indigo）

**UI 布局示意**:
```
┌─────────────────────────────────────┐
│  输入小说背景描述                      │
│  ┌─────────────────────────────────┐ │
│  │ 简短描述（必填）                  │ │
│  │ [textarea]                      │ │
│  └─────────────────────────────────┘ │
│  ┌─────────────────────────────────┐ │
│  │ 故事走向偏好                      │ │
│  │ [select]                        │ │
│  └─────────────────────────────────┘ │
│                                     │
│  [开始生成] [AI 随机生成]              │
└─────────────────────────────────────┘
```

#### 3.1.2 交互逻辑

**点击"随机生成"按钮**:
1. 无需检查描述输入框是否为空
2. 直接开始 SSE 流式生成
3. 显示生成状态指示（复用现有）
4. 生成完成后显示确认对话框（复用现有）

**按钮状态**:
- idle 状态：可点击
- streaming 状态：显示"停止生成"按钮（复用现有逻辑）
- connecting 状态：显示加载动画（复用现有）

#### 3.1.3 请求参数

**SSE 请求**:
```
POST /api/generation/world-setting/stream
Body:
{
  "novel_id": novelId,
  "random_generate": true,
  "genre": "都市职场"  // MVP 固定为都市职场
}
```

---

### 3.2 后端需求

#### 3.2.1 API 变更

**接口**: `/api/generation/world-setting/stream`

**新增参数**:
- `random_generate`: boolean，是否随机生成
- `genre`: string，小说类型（MVP 固定接收"都市职场"）

**行为变更**:
- 当 `random_generate=true` 时，忽略 `user_description` 参数
- 调用新的随机生成 Prompt 构建方法

#### 3.2.2 服务层变更

**文件**: `generation_service.py`

**新增方法**:
- `_build_random_world_setting_prompt(genre: str)` - 构建随机生成 Prompt
- `stream_random_generate_world_setting(novel_id, genre)` - 随机生成流式方法

**修改方法**:
- `stream_generate_world_setting()` - 新增对 `random_generate` 参数的处理

#### 3.2.3 随机生成 Prompt 设计

**Prompt 特点**:
1. 明确指定小说类型为"都市职场"
2. 要求 AI 自行决定所有设定细节（时代背景、人物设定、冲突等）
3. 输出结构不变，仍为完整的 `world_setting.json` 格式
4. 复用现有的写作风格指南

**Prompt 核心内容**:
```
# 任务：随机生成都市职场小说世界观

## 小说类型
都市职场

## 输出要求
请随机生成一个完整的都市职场小说世界观设定。

### 主题选择（随机选择其一）
- 职场晋升：主角从基层员工晋升为高管
- 商业博弈：主角在商业竞争中突围
- 创业奋斗：主角从零开始创业
- 职场爱情：主角在职场中遇到爱情

### background（背景设定）
- era: "现代都市"
- era_name: 具体时代名称（如"2024年上海"）
- geography: 都市地理设定，包含：
  - world_name: 城市名称
  - regions: 至少3个主要区域（如商务区、住宅区、工业区）
- society: 职场社会设定，包含：
  - power_structure: 企业权力结构
  - social_classes: 职场层级（如高管、中层、基层）
  - key_institutions: 关键企业/机构

### power_system（能力体系 - 职场改编）
- name: "职场等级体系"
- levels: 职场等级列表（如实习生→专员→主管→经理→总监→高管）
- key_rules: 职场晋升规则

### core_conflict（核心冲突）
- main_conflict: 职场主线冲突（如晋升竞争、商业博弈、职场阴谋）
- sub_conflicts: 子冲突列表（如感情纠葛、家庭压力）

### special_elements（特殊元素）
至少包含2个特殊元素（如关键项目、重要资源、特殊关系）

## 约束条件
1. 设定必须符合都市职场现实逻辑
2. 冲突必须能支撑至少50章的职场剧情发展
3. 人物设定需要有明确的职业背景和目标

## 输出格式
输出严格遵循 JSON 结构...
```

## 4. 技术约束

### 4.1 不修改的部分

| 组件 | 原因 |
|------|------|
| SSE 客户端 (`sse-client.ts`) | 复用现有流式处理逻辑 |
| JSON 解析逻辑 (`tryParseJSON`) | 复用现有解析逻辑 |
| 确认对话框 | 复用现有 UI 组件 |
| 世界观展示组件 (`WorldSettingViewer`) | 复用现有展示逻辑 |

### 4.2 需修改的部分

| 文件 | 修改内容 | 工作量 |
|------|----------|--------|
| `WorldBuilder.tsx` | 新增按钮和交互逻辑 | 1-2 小时 |
| `api_novels.py` | 新增随机生成参数处理 | 0.5 小时 |
| `generation_service.py` | 新增随机生成 Prompt 方法 | 1-2 小时 |

### 4.3 向后兼容

- 现有"描述输入 → 生成"流程完全不变
- 随机生成作为新增功能，不影响现有 API 调用

## 6. 用户故事

### 用户故事 1: 无灵感用户

```markdown
**作为** 一个完全没有创作灵感的用户
**我希望** 点击一个按钮就能获得一个完整的都市世界观
**以便** 我有一个创作起点，可以在此基础上调整和创作

**验收标准**:
- 点击按钮后 30 秒内开始显示生成内容
- 生成完整的世界观设定（背景、体系、冲突、元素）
- 可以在生成结果基础上编辑和保存
```

### 用户故事 2: 快速体验用户

```markdown
**作为** 一个想快速体验系统的新用户
**我希望** 不需要构思内容就能体验完整流程
**以便** 快速了解系统功能和生成质量

**验收标准**:
- 从点击到获得结果不超过 60 秒
- 生成的世界观质量与手动输入相当
- 可以继续体验后续的大纲生成流程
```

| 维度 | 评分 (1-5) | 说明 |
|------|-----------|------|
| **R**each | 3 | 影响部分用户（无灵感用户） |
