## 1. 需求背景

### 问题描述

灵笔项目 AI 生成的所有内容存在以下质量问题，导致内容难以阅读、无法直接使用：

| 问题类型 | 具体表现 | 影响 |
|---------|---------|------|
| **信息密度过高** | 一句话包含5-7个核心信息点 | 阅读负担重，难以理解 |
| **术语堆砌** | 一句话出现6-8个专业术语 | 读者无法快速消化概念 |
| **缺乏过渡衔接** | 从A到B的因果关系无铺垫 | 逻辑跳跃，叙事不连贯 |
| **句子结构复杂** | 多层并列结构叠加 | 长句过长，节奏单调 |
| **风格不统一** | 有时文言、有时白话混用 | 风格割裂，阅读体验差 |

### 问题示例

**原文（大纲概述）**：
```
他摒弃武道速成幻梦，用战术思维重构蛊术逻辑、用战场感知解构瘴毒规律，
在士族监察使突袭、峒寨血祭危机与灵枢石走私案中连续破局，
首次验证'技术可补境界'——以自制'震频骨笛'干扰蛊虫神经节，瓦解守碑派伏杀。
本卷终结于他持半卷《百峒图》闯入雾瘴谷，成为百年来首位未服醒窍蛊而通关通窍境试炼的'异类'。
```

**分析**：
- 第一句：4个核心信息点 + 4个术语
- 第二句：3个事件并列 + 2个术语
- 第三句：1个概念验证 + 1个工具 + 2个术语 + 1个结果
- 第四句：1个结尾 + 2个术语 + 1个成就
- **总计：约100字包含11个信息点、8个术语，无过渡衔接**

### 用户痛点

| 痛点 | 影响 | 严重度 |
|------|------|--------|
| 生成内容无法直接使用 | 需要大量人工修改 | **P0** |
| 阅读理解困难 | 影响创作效率和决策 | **P0** |
| 所有阶段都受影响 | 世界观、大纲、正文全部质量差 | **P0** |

## 3. 详细需求清单

### 3.1 Prompt 风格约束优化

#### 3.1.1 写作风格指南（新增）

**世界观生成 Prompt 新增**：
```
## 写作风格指南（强制遵守）

### 信息密度控制
1. 每句话不超过 2 个核心信息点
2. 专业术语每段不超过 3 个，首次出现需有简短解释
3. 描述类内容：先总述（1句），再分述（每点1句）

### 句子结构控制
1. 单句长度不超过 40 字
2. 长短句交替：每 2 个短句后可接 1 个中长句（不超过 60 字）
3. 避免多层并列：并列结构不超过 3 层

### 过渡衔接控制
1. 因果关系必须明确过渡（如"因此"、"于是"、"这导致"）
2. 时间顺序必须标注（如"随后"、"最终"、"在此期间"）
3. 场景切换必须说明（如"与此同时"、"另一处"）

### 术语处理规则
1. 世界观术语首次出现需简短定义（不超过 15 字）
2. 能力等级术语需用"→"标注晋升路径
3. 地名/组织名需标注所属层级（如"中州·承天阙"）
```

#### 3.1.2 大纲生成 Prompt 新增

```
## 写作风格指南（强制遵守）

### arc_summary 概述要求
1. 字数限制：每卷概述不超过 150 字
2. 结构要求：起因（1句）→ 发展（2句）→ 结局（1句）
3. 事件密度：每句最多 1 个核心事件
4. 术语限制：每卷概述最多 3 个术语

### 章节规划要求
1. key_events：每章最多 3 个事件，每个事件不超过 20 字
2. turning_points：每个转折点需包含 event + impact（各不超过 20 字）
3. 标题要求：章节标题不超过 15 字，需有画面感

### 伏笔计划要求
1. hint：伏笔提示不超过 30 字，需有具体画面
2. 回收时机：间隔不超过 50 章
```

#### 3.1.3 章节正文生成 Prompt 新增

```
## 写作风格指南（强制遵守）

### 正文叙事要求
1. 段落长度：每段 100-200 字，最多不超过 300 字
2. 句子节奏：短句（10-20字）占 60%，中句（20-40字）占 30%，长句（40-60字）占 10%
3. 信息密度：每段最多 3 个核心信息点

### 对话描写要求
1. 对话占比：不超过正文 30%
2. 对话长度：单句对话不超过 50 字
3. 对话+动作：每段对话需配 1 句动作或神态描写

### 场景描写要求
1. 首次出场场景：不超过 100 字的环境描写
2. 已有场景：不超过 50 字的过渡描写
3. 战斗场景：不超过 200 字，分 2-3 段

### 情感描写要求
1. 情感铺垫：重大情感转折前需有 1-2 段铺垫
2. 情感表达：避免直接说"他很愤怒"，用动作/神态/心理活动表现
3. 情感过渡：情感变化需有触发事件说明
```

### 3.2 信息密度控制规则

#### 3.2.1 量化指标

| 内容类型 | 最大信息密度 | 最大术语密度 | 最大句长 |
|---------|-------------|-------------|---------|
| 世界观背景描述 | 2 信息点/句 | 3 术语/段 | 40 字 |
| 能力体系描述 | 2 信息点/句 | 2 术语/段 | 40 字 |
| 大纲 arc_summary | 1 事件/句 | 3 术语/卷 | 50 字 |
| 章节标题 | 1 画面 | 0 术语 | 15 字 |
| 正文段落 | 3 信息点/段 | 2 术语/段 | 200 字（段落） |

#### 3.2.2 Prompt 中的检查指令

新增 Prompt 尾部检查指令：
```
## 输出前自检（强制执行）

生成内容后，请按以下规则自检：

1. 【信息密度】每句话是否不超过 2 个核心信息点？
   - 若超标，请拆分为多句

2. 【术语密度】每段是否不超过 3 个专业术语？
   - 若超标，请将术语分散到不同段落

3. 【句子长度】单句是否不超过 40 字？
   - 若超标，请拆分或重新组织

4. 【过渡衔接】因果关系是否有明确过渡词？
   - 若缺失，请添加"因此/于是/这导致"

5. 【风格统一】是否全程保持一致的语言风格？
   - 若混用，请统一为一种风格

请确保以上检查全部通过后再输出。
```

### 3.3 内容质量验证层（P1）

#### 3.3.1 后端验证模块

新增 `backend/src/domain/services/content_quality_service.py`：

| 验证规则 | 检查方法 | 处理方式 |
|---------|---------|---------|
| 信息密度超标 | 统计句子信息点数 | 标记警告，不强制修改 |
| 术语密度超标 | 统计段落术语数 | 标记警告 |
| 句子长度超标 | 统计单句字数 | 标记警告 |
| 过渡衔接缺失 | 检查因果过渡词 | 标记建议 |
| 风格不统一 | 检查文言/白话混用 | 标记建议 |

#### 3.3.2 前端质量提示

在生成结果展示时，显示质量评分：

```
质量评分：85/100
- 信息密度：✅ 通过
- 术语密度：⚠️ 第3段超标（建议分散）
- 句子长度：✅ 通过
- 过渡衔接：⚠️ 缺少2处过渡
- 风格统一：✅ 通过
```

#### 3.4.1 可配置项

| 配置项 | 选项 | 默认值 |
|--------|------|--------|
| 语言风格 | 古典文言 / 现代白话 / 混合风格 | 现代白话 |

  }
}

## 4. 技术约束

### 4.1 Prompt 修改位置

| 文件 | 修改方法 | 影响 |
|------|---------|------|
| `generation_service.py` | `_build_world_setting_prompt()` | 世界观生成 |
| `generation_service.py` | `_build_outline_prompt()` | 大纲生成 |
| `generation_service.py` | `_build_chapter_prompt()` | 章节正文 |

### 4.2 不修改的部分

- AI Provider 层（`ai_provider.py`）：保持不变
- API 接口层：保持不变
- 前端展示层：保持不变（P1/P2 新增提示）

### 4.3 回滚机制

若新 Prompt 导致生成失败率上升：
1. 保留旧 Prompt 作为 fallback
2. 提供 Prompt 版本切换开关

## 6. 优先级评估

### RICE 评分

| 维度 | 评分 (1-5) | 说明 |
|------|-----------|------|
| **R**each | 5 | 影响所有用户，所有生成阶段 |
| **I**mpact | 5 | 直接决定产品可用性 |
| **C**onfidence | 5 | 用户已确认问题严重 |
| **E**ffort | 2 | 主要修改 Prompt，工作量较小 |

**Priority = (5 × 5 × 5) / 2 = 62.5** → **最高优先级 P0**
