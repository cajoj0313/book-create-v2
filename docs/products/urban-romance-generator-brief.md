## 1. 产品转型背景

### 原定位
- AI 主导的多类型小说生成平台（武侠/玄幻/都市等）
- 完整的创作流程（世界观→大纲→章节→校验）
- 复杂的状态追踪（时间线/人物状态/伏笔）
- 36 条校验规则（逻辑/语法/风格/人物/伏笔/商业/感情）

### 新定位
- **仅支持都市言情小说**
- 简化创作流程（世界观→大纲→章节）
- 移除复杂状态追踪
- 简化校验规则（仅保留基础语法和感情线校验）

### 转型原因

| 问题 | 影响 | 解决方案 |
|------|------|----------|
| 功能过于复杂 | 用户只想做个"爽文生成器" | 简化为核心生成功能 |
| 状态追踪繁琐 | 都市言情不需要复杂伏笔/时间线 | 移除状态追踪 |
| 校验规则过多 | 影响生成效率 | 简化为基础校验 |

## 2. 都市言情小说特点

### 核心要素

| 元素 | 说明 | 生成要求 |
|------|------|----------|
| **男主设定** | 高富帅/霸道总裁/暖男/腹黑 | 随机生成或用户指定 |
| **女主设定** | 白富美/职场新人/小透明 | 随机生成或用户指定 |
| **感情线** | 遇见→误会→升温→表白→波折→甜蜜 | 必须有完整的感情节奏 |
| **职场线** | 晋升/创业/商业博弈 | 作为感情线的背景和催化剂 |
| **爽点** | 打脸/逆袭/表白/求婚/甜蜜互动 | 每 3-5 章一个爽点 |

### 章节结构（标准都市言情）

```
第 1-3 章：相遇（误会/冲突）
第 4-10 章：职场交集，感情萌芽
第 11-20 章：误会解除，感情升温
第 21-30 章：表白，甜蜜互动
第 31-40 章：波折（情敌/家庭/工作）
第 41-50 章：解决波折，求婚/结局
```

### 感情节奏表

| 章节范围 | 感情阶段 | 状态描述 | 爽点类型 |
|----------|----------|----------|----------|
| 1-3 | 初遇/误会 | 冷淡/敌视 | 打脸女主 |
| 4-10 | 暧昧/试探 | 好感萌芽 | 偶遇救场 |
| 11-20 | 升温/表白 | 甜蜜暧昧 | 表白/亲吻 |
| 21-30 | 甜蜜/互动 | 恋人状态 | 约会/送礼 |
| 31-40 | 波折/分离 | 矛盾冲突 | 挽留/解释 |
| 41-50 | 和解/结局 | 甜蜜结局 | 求婚/婚礼 |

## 3. MVP 功能清单

### 保留功能

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 世界观生成 | P0 | 简化为都市言情模板 |
| 大纲生成 | P0 | 简化为感情节奏表 |
| 章节续写 | P0 | 聚焦感情线和爽点 |
| 随机生成 | P0 | 一键生成完整言情世界观 |
| 基础人物管理 | P1 | 男主/女主/配角增删 |
| 小说管理 | P1 | 创建/删除/列表 |

### 移除功能

| 功能 | 原优先级 | 移除原因 |
|------|----------|----------|
| 时间线状态追踪 | P0 | 都市言情不需要复杂时间线 |
| 人物状态追踪 | P0 | 简化为感情阶段标记 |
| 伏笔管理 | P0 | 都市言情伏笔简单，不需要管理 |
| 36条校验规则 | P0 | 简化为 5 条基础规则 |
| 版本历史 | P1 | 简化版可暂时不做 |
| WorldBuilder 增删子项 | P1 | 使用固定模板 |

### 简化校验规则（5 条）

| 规则ID | 规则名称 | 说明 |
|--------|----------|------|
| G001 | 错别字检查 | 自动修正 |
| G002 | 标点错误 | 自动修正 |
| E001 | 感情节奏检查 | 检查感情阶段是否符合节奏表 |
| E002 | 爽点密度检查 | 每 5 章至少 1 个爽点 |
| P001 | 人物称呼一致 | 男主/女主称呼固定 |

## 5. 用户故事

### 主用户故事

```markdown
**作为** 一个想写都市言情小说的用户
**我希望** 点击几个按钮就能生成一本言情爽文
**以便** 快速获得阅读乐趣或分享给朋友

**验收标准**:
- 随机生成可获得完整的言情世界观
- 大纲自动按感情节奏表生成
- 章节聚焦感情线和爽点
- 无需复杂配置，一键开始
```
