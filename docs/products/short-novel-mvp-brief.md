## 1. 产品概述

### 1.1 产品定位

**灵笔短篇小说生成器** - 一键生成 10-15 章完整短篇小说的 AI 创作工具

**目标用户**: 想要快速完成一部完整小说的创作者，而非长篇连载作者

**核心价值**: 
- 2 小时内完成一部 2-3 万字的完整短篇小说
- 故事有完整起承转合，不是碎片化内容
- 人物有深度，情节有因果，主题有探讨

---

### 1.2 为什么先做短篇？

| 维度 | 长篇 (100-200 章) | 短篇 (10-15 章) | MVP 选择 |
|------|------------------|----------------|----------|
| **生成耗时** | 4-8 小时 | 30-60 分钟 | ✅ 短篇 |
| **内容验证** | 难验证完整性 | 易验证完整性 | ✅ 短篇 |
| **用户耐心** | 易中途放弃 | 易完成 | ✅ 短篇 |
| **技术复杂度** | 多线叙事管理复杂 | 单线为主，简单 | ✅ 短篇 |
| **迭代速度** | 慢 | 快 | ✅ 短篇 |

**策略**: 短篇验证成功后，扩展成长篇（同一架构，增加多线叙事管理）

## 2. 用户故事

### 2.1 核心用户故事

```
作为想要快速创作完整小说的读者/写手，
我希望输入一个简单的故事想法（如"霸道总裁爱上我"），
就能自动生成一部 10-15 章的完整短篇小说，
以便快速体验创作完整故事的成就感。
```

### 2.2 细分用户故事

| 用户故事 | 验收标准 | 优先级 |
|----------|----------|--------|
| **US1**: 我想输入故事想法后，AI 帮我生成世界观 | 世界观包含城市、职场、男女主设定、感情线、核心冲突 | P0 |
| **US2**: 我想基于世界观生成大纲 | 大纲包含 10-15 章规划，每章有核心事件和感情阶段 | P0 |
| **US3**: 我想看到完整故事梗概后再拆分章节 | 故事梗概 3000-5000 字，涵盖所有章节 | P0 |
| **US4**: 我想批量生成章节，而不是逐章生成 | 一次生成 3-5 章，章节之间有因果联系 | P0 |
| **US5**: 我想在生成过程中看到进度 | 有进度条显示当前处于哪一步 | P1 |
| **US6**: 我想编辑生成后的内容 | 可编辑世界观、大纲、章节内容 | P1 |
| **US7**: 我想导出小说为文本文件 | 可导出 TXT/EPUB 格式 | P2 |

## 3. 功能需求

### 3.1 功能架构图

```
短篇小说生成器 (Short Novel Generator)
├── 世界观生成 (World Building)
│   ├── 基础设定（城市、职场）
│   ├── 人物设定（男主、女主、配角 2-3 人）
│   ├── 感情线（5-8 个阶段）
│   ├── 核心冲突（1-2 个）
│   └── 主题定义（1 个核心主题）
│
├── 大纲生成 (Outline Generation)
│   ├── 章节规划（10-15 章）
│   ├── 每章核心事件（50-80 字，有因果链）
│   ├── 感情阶段标注
│   └── 关键场景预设
│
├── 故事梗概 (Story Synopsis)
│   ├── 完整故事（3000-5000 字）
│   ├── 关键情节节点（5-8 个）
│   └── 因果链说明
│
├── 章节拆分 (Chapter Splitting)
│   ├── 批量拆分（每次 3-5 章）
│   ├── 章节连贯性检查
│   └── 章节编辑与调整
│
└── 导出功能 (Export)
    ├── TXT 导出
    └── EPUB 导出（P2）
```

---

### 3.2 短篇小说规格定义

| 规格项 | 值 | 说明 |
|--------|-----|------|
| **章节数** | 10-15 章 | 用户可选 10/12/15 章 |
| **总字数** | 2-3 万字 | 每章 2000-3000 字 |
| **故事线** | 单线为主（爱情线 80% + 事业线 20%） | MVP 不做复杂多线 |
| **人物数** | 主角 2 人 + 配角 2-4 人 | 共 4-6 人 |
| **感情阶段** | 5-6 个阶段 | 初遇→暧昧→升温→甜蜜→波折→结局 |
| **核心冲突** | 1-2 个 | 外部冲突 + 内心冲突 |
| **主题** | 1 个核心主题 | 如"爱与成长" |

**需求描述**: 生成完整故事梗概（3000-5000 字）

## 5. 数据结构

### 5.1 核心数据模型

```typescript
// 短篇小说
interface ShortNovel {
  novel_id: string
  title: string
  genre: '都市言情' | '武侠' | '奇幻'  // MVP 仅支持都市言情
  target_chapters: 10 | 12 | 15  // 目标章节数
  total_word_count: number  // 总字数
  status: 'draft' | 'completed'
  created_at: string
  updated_at: string
}

// 世界观（短篇小说简化版）
interface WorldSetting {
  novel_id: string
  background: {
    city: string
    workplace: string
    workplace_name: string
  }
  male_lead: Character  // 男主
  female_lead: Character  // 女主
  supporting_chars: SupportingCharacter[]  // 配角（2-4 人）
  emotion_arc: {
    stages: string[]  // 感情阶段（5-6 个）
    milestone_events: MilestoneEvent[]  // 里程碑事件
  }
  main_conflict: {
    type: string
    description: string
  }
  theme: {
    main: string
    description: string
  }
}

// 人物
interface Character {
  name: string
  age: number
  identity: string
  occupation: string
  appearance: string
  personality: string[]
  inner_wound: string  // 内心创伤
  growth_arc: string  // 成长弧光
}

// 大纲
interface Outline {
  novel_id: string
  chapters: ChapterOutline[]  // 10-15 章
}

interface ChapterOutline {
  chapter_num: number
  title: string
  key_events: string[]  // 核心事件（2-3 个，50-80 字/事件）
  emotion_stage: string
  emotion_progress: string
  location: string
  word_count_target: number
}

// 故事梗概
interface StorySynopsis {
  novel_id: string
  story_content: string  // 3000-5000 字
  key_plot_points: PlotPoint[]
}

interface PlotPoint {
  point: number
  chapter_range: string
  event: string
}

// 章节
interface Chapter {
  novel_id: string
  chapter_num: number
  title: string
  content: string  // 2000-3000 字
  word_count: number
  emotion_stage: string
  summary: string
}
```

## 6. 技术约束

### 6.3 后端约束

| 约束 | 说明 |
|------|------|
| **存储** | 文件存储（JSON），不做数据库 |
