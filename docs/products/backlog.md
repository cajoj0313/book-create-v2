### FEAT-17: 章节大纲关联与空白状态优化 ⭐⭐ 新需求（P1）

**用户故事**: 作为小说创作者，我希望AI生成章节内容时能参考大纲的事件和感情阶段设定，以便生成的内容与我规划的故事走向一致；同时希望章节未生成时页面显示简洁提示，而不是醒目的警告。

**核心诉求**: 
1. 大纲上下文注入到AI Prompt（key_events、emotion_stage）
2. 章节未生成时去掉warning样式，显示简洁提示

**验收标准**:
- [ ] 后端 `_build_chapter_prompt()` 注入大纲上下文
- [ ] 后端 API 模型支持 `outline_context` 参数
- [ ] 前端传递 `outline_context`（已实现，需后端接收）
- [ ] AI生成内容覆盖 60%+ 大纲事件
- [ ] 空白状态去掉 warning 样式

**RICE Priority**: (4 × 4 × 4) / 0.5 = **128** → 高优先级 P1

**排期计划**:
| 任务 | 工时 | 优先级 |
|------|------|--------|
| 后端API模型新增参数 | 0.5h | P1 |
| 后端Prompt注入大纲 | 1h | P1 |

| pytest 测试验证 | 0.5h | P1 |
| 人工效果验证 | 1h | P1 |
| **总计** | **4h** | - |
