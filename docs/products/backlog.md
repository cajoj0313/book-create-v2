## P0 - 核心缺失（必须完成）

### FEAT-01: 状态追踪 API

**用户故事**: 作为作者，我希望能手动维护时间线/人物状态/伏笔状态，以便补录 AI 遗漏的信息

**验收标准**:
- [ ] 时间线 CRUD API（GET/PUT/POST/DELETE）
- [ ] 人物状态 CRUD API（GET/PUT）
- [ ] 伏笔状态 CRUD API（GET/PUT）
- [ ] 所有 API 返回标准格式 `{success: true, data: {...}}`
- [ ] 单元测试覆盖率 ≥ 80%

**优先级评分**:
| 维度 | 评分 (1-5) | 说明 |
|------|-----------|------|
| 业务价值 | 5 | 状态追踪是小说创作核心功能 |
| 用户痛点 | 5 | AI 遗漏时无法手动补救 |
| 技术可行性 | 4 | 后端 API 增加较简单 |
| 依赖风险 | 2 | 无依赖 |

**MVP 范围**: Must have（全部实现）
