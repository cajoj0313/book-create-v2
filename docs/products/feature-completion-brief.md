## 1. 需求背景

### 问题描述

灵笔项目当前已实现核心创作流程（世界观 → 大纲 → 章节 → 校验），但存在 **57 项功能缺失**，主要集中在：

1. **"新增不能删"问题** - 创建后无法删除（小说、人物、章节、伏笔等）
2. **状态追踪缺失** - 生成后的时间线/人物状态/伏笔状态无法手动维护
3. **增删功能缺失** - 世界观、大纲中无法新增/删除子项
4. **版本历史缺失** - 无法查看历史版本或回滚

### 用户痛点

| 痛点 | 影响 | 严重度 |
|------|------|--------|
| 人物创建后无法删除 | 人物库积累废弃人物，影响一致性校验 | **P0** |
| 伏笔无法手动更新状态 | AI 生成遗漏时无法补救，影响伏笔追踪 | **P0** |
| 版本无法回滚 | 误操作或 AI 生成质量差时无法恢复 | **P1** |
| 世界观子项无法增删 | 无法扩展设定，限制创作灵活性 | **P1** |

## 3. 详细需求清单

### 3.1 P0 - 核心缺失（必须完成）

#### 3.1.1 状态追踪 API（后端）

| API | 方法 | 功能 | 验收标准 |
|-----|------|------|----------|
| `/novels/{id}/state/timeline` | GET | 获取时间线 | 返回完整时间线 JSON |
| `/novels/{id}/state/timeline` | PUT | 更新时间线 | 支持添加/删除/修改事件 |
| `/novels/{id}/state/timeline/events` | POST | 添加时间线事件 | 新事件自动排序 |
| `/novels/{id}/state/timeline/events/{order}` | DELETE | 删除时间线事件 | 删除后重排序 |
| `/novels/{id}/state/character-states` | GET | 获取人物状态 | 返回所有人物当前状态 |
| `/novels/{id}/state/character-states` | PUT | 批量更新人物状态 | 支持多人物同时更新 |
| `/novels/{id}/state/character-states/{char_id}` | PUT | 更新单人物状态 | 支持位置/情绪/能力更新 |
| `/novels/{id}/state/foreshadowing` | GET | 获取伏笔状态 | 返回伏笔列表及统计 |
| `/novels/{id}/state/foreshadowing` | PUT | 批量更新伏笔状态 | 支持多伏笔同时更新 |
| `/novels/{id}/state/foreshadowing/{fs_id}` | PUT | 更新单伏笔状态 | 支持标记已埋/已回收 |

**验收标准**:
- [ ] 所有 API 返回 `{success: true, data: {...}}` 格式
- [ ] 单元测试覆盖率 ≥ 80%
- [ ] API 文档已更新（Swagger）

## 4. 技术约束

### 4.1 API 设计规范

所有新增 API 必须遵循：

```json
// 成功响应
{
  "success": true,
  "data": { ... }
}

// 失败响应
{
  "success": false,
  "error": "错误信息"
}
```

### 4.2 数据存储规范

- 时间线文件：`data/novels/{id}/state/timeline.json`
- 人物状态文件：`data/novels/{id}/state/character_states.json`
- 伏笔状态文件：`data/novels/{id}/state/foreshadowing.json`
- 版本历史：`data/novels/{id}/chapters/chapter_xxx_v{n}.json`

### 4.3 前端组件规范

- 弹窗使用 `dialog-overlay` + `dialog-paper` 类
- 删除确认弹窗必须包含"取消"和"确认"按钮
- Toast 提示使用 `toast-success` / `toast-error` / `toast-warning`
